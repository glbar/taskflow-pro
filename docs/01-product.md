## 페르소나

| 항목 | 내용 |
|------|------|
| 대상 | 10명 규모 스타트업 팀리더 |
| 연령 | 30~40대 |

## MVP 범위

### CRUD 4종 — 모두 화면에서 동작

| 동작 | 설명 |
|------|------|
| 추가 (Create) | 태스크 제목·상태·마감 시각 입력 후 생성 |
| 목록 (Read) | 전체 태스크 목록 조회, 상태별 필터 |
| 수정 (Update) | 제목·상태·마감 시각 인라인 수정 |
| 삭제 (Delete) | 태스크 삭제 (확인 모달) |

### 상태 분류

```
Todo  →  In Progress  →  Done
```

### 마감 시각 (due_at)

- **날짜 + 시간** 모두 지정 (예: `2026-05-12 18:00`)
- 날짜만 입력하는 방식 사용 안 함

### UI

- 라이트 / 다크 테마 토글 — `localStorage` 에 저장, 새로고침 유지
- 모바일 반응형 — 최소 **360px** 기준 레이아웃 깨짐 없음
- 톤: **macOS 스타일** — 둥근 모서리, 부드러운 그림자, 반투명 카드, 시스템 폰트

### 실시간

- MVP 단계: **폴링(polling)** 방식
- WebSocket은 확장 단계에서 도입, MVP에서 사용 안 함

---

## 성공 기준

MVP 완료 조건은 아래 5가지를 **모두** 충족해야 한다.

| # | 기준 | 측정 방법 |
|---|------|----------|
| 1 | 새로고침 후 데이터 유지 | DB 저장 확인 |
| 2 | 360px에서 레이아웃 깨짐 없음 | Chrome DevTools 360px 시뮬레이션 |
| 3 | API 응답 200ms 이하 | 네트워크 탭 측정 |
| 4 | CRUD 4종 화면에서 동작 | 직접 시나리오 실행 |
| 5 | 라이트/다크 테마 토글 작동 | 새로고침 후 테마 유지 확인 |

---

## 확장 로드맵 (MVP 이후)

아래 기능은 MVP 완료 후 순차적으로 검토한다.

```
JWT 로그인 → 팀 관리 → Kanban 보드 → 채팅 → CI/CD 자동화 → WebSocket
```
